Camp Swing Outdoor

운영자용 Google 스프레드시트 가이드
버전 1.0 · 2026

1. 시작하기 전에

Camp Swing Outdoor 사이트는 코드를 수정하지 않아도 Google 스프레드시트만 편집하면 화면이 바뀌도록 설계되었습니다.
이 매뉴얼은 처음 운영을 맡으신 분도 시트만 잘 관리하면 사이트 콘텐츠를 자유롭게 운영하실 수 있도록 안내합니다.

운영자가 직접 하는 일
① 시트 안의 텍스트(이벤트 이름·날짜·계좌·일정·라인업 등) 수정
② 사용자가 제출한 신청 명단 확인
③ 새 행사 시작 시 시트 내용 갱신

전체 구조 한눈에 보기

사이트는 2개의 Google 스프레드시트를 사용합니다.

스프레드시트역할운영자의 작업
폼 신청 시트 (server response) 사용자가 신청 폼에서 보낸 내용이 자동으로 쌓이는 곳 직접 입력하지 않고 명단을 확인하는 용도
서버 세팅 시트 (server settings) 사이트에 보이는 텍스트·일정·라인업 등을 보관 운영자가 직접 편집 (이 매뉴얼의 핵심 영역)
💡 두 시트의 ID(주소)는 시스템 담당자가 서버에 설정합니다. 운영자는 이미 설정된 시트의 내용만 관리합니다.

2. 폼 신청 시트 (server response)

사용자가 사이트에서 [신청 제출]을 누르면 이 시트의 마지막 줄에 새 행이 자동으로 추가됩니다.

⚠️ 절대 하지 마세요
운영자가 신청 시트에 직접 행을 추가하거나 기존 행을 수정하지 마세요.
사용자 입력만 자동으로 쌓이도록 되어 있어, 수동 편집은 데이터 정합성을 깨뜨릴 수 있습니다.

시트 입력의 공통 규칙

  • 1행은 자유 — 한글 라벨을 자유롭게 사용하셔도 됩니다 (예: A1="키", B1="값"). 시스템은 1행을 무시합니다.
  • 실제 데이터는 2행부터 입력하세요.
  • 탭 이름과 키(영문)는 정확히 일치해야 합니다 — 대소문자, 공백까지.

4-14. 프로필 이미지 적용 방법 (DJ / 강사 / 스태프 / Invited Dancers 공통)

각 라인업 탭의 프로필 이미지 경로 칸(Dj=E열, 나머지=D열)에 이미지 주소를 넣으면 카드에 사진이 표시됩니다. 두 가지 방법이 있습니다.

방법 ① 사이트에 직접 올린 이미지 (권장 · 안정적)

  • 이미지 파일을 프로젝트의 src/main/resources/static/img/ 폴더에 넣어 배포합니다. (개발/운영 담당자에게 요청)
  • 시트에는 /img/파일명.jpg 형태의 경로를 입력합니다. 예: /img/dancers/joje.jpg
  • 외부 사이트에 의존하지 않아 가장 안정적입니다.

방법 ② 외부 이미지 호스팅 (imgbb 등) — 직접 링크 사용

⚠️ "공유 페이지 주소"가 아니라 "직접 이미지 링크"를 넣어야 합니다
imgbb에서 복사되는 https://ibb.co/XXXX 는 이미지를 보여주는 웹페이지 주소라 이미지로 표시되지 않습니다. 실제 이미지 파일 주소는 i. 로 시작하고 .jpg / .png 등으로 끝납니다.
구분예시사용
공유 페이지 (뷰어)https://ibb.co/9mHVZH9Y❌ 표시 안 됨
직접 이미지 링크https://i.ibb.co/HLTxhTDg/photo.jpg✅ 이것을 입력

직접 링크 찾는 법

  • imgbb 업로드 후 링크 칸의 드롭다운을 "Direct links" 로 바꾸면 i.ibb.co/....jpg 형태가 바로 나옵니다.
  • 또는 imgbb가 제공하는 HTML / BBCode 코드 안에서 <img src="..."> 의 src 값(= [img] 안쪽 주소)을 복사합니다.
  • 또는 미리보기 이미지에서 마우스 우클릭 → "이미지 주소 복사".

구분 기준: 주소에 i. 가 붙어 있고(i.ibb.co ✅ / ibb.co ❌), 끝이 .jpg·.png·.webp 같은 이미지 확장자인지 확인하세요.

5. 꼭 지켜야 할 5가지 원칙

1. 탭 이름을 한글로 바꾸지 마세요
탭 이름은 시스템이 영문 이름으로만 인식합니다. Event를 이벤트로 바꾸면 사이트가 그 탭을 찾지 못합니다.
2. 열(컬럼) 순서를 바꾸지 마세요
시스템은 "1열 = 신청번호, 2열 = 일시..." 식으로 위치 기반으로 읽습니다. 한 칸이라도 밀리면 내용이 잘못 표시됩니다.
3. 신청 시트에 손으로 행을 추가하지 마세요
신청 시트는 사용자 폼 입력 전용입니다. 직접 입력하면 데이터 정합성이 깨질 수 있어요.
4. 영문 키(A열)는 정확히 입력하세요
name을 Name 또는 name (공백 포함)으로 입력하면 인식되지 않습니다. 대소문자·공백을 정확히 맞춰야 합니다.
5. 서비스 계정 권한을 빼지 마세요
사이트는 "서비스 계정"이라는 이메일로 시트를 읽고 씁니다. 시트 공유 설정에서 이 이메일의 권한을 제거하면 사이트가 시트와 통신하지 못합니다.
· 신청 시트: 편집자 권한 필수
· server settings 시트: 뷰어 권한 권장

6. 자주 하는 작업 빠르게 찾기

Q. 행사 날짜를 바꾸고 싶어요
Settings 시트 → Event 탭 → startDate, endDate 값 수정
Q. 입금 계좌를 바꾸고 싶어요
Settings 시트 → Event 탭 → bankName, bankAccount, accountHolder 값 수정
Q. 문의 이메일을 바꾸고 싶어요
Settings 시트 → Event 탭 → contactEmail 값 수정
Q. 행사 일정을 수정하고 싶어요
Settings 시트 → Schedule 탭에서 행 추가/삭제/수정
Q. 픽업버스 시간을 변경하고 싶어요
Settings 시트 → PickupBus 탭에서 행 수정
Q. DJ를 새로 추가하고 싶어요
Settings 시트 → Dj 탭에서 새 행 추가 (id·스테이지명·이미지경로·소개·순서 입력)
Q. 신청자 명단을 보고 싶어요
① 사이트의 [신청확인] 버튼을 통해 보거나, ② 신청 시트의 해당 탭(PartyPass/Campsite/Dormitory)을 직접 열어 확인
Q. 사이트에 바뀐 내용이 안 보여요
설정에 따라 변경 반영에 최대 10분이 걸릴 수 있습니다. 잠시 후 다시 확인하세요. 그래도 안 되면 시스템 담당자에게 문의하세요.
Q. 시트의 1행에 한글로 라벨을 적어도 되나요?
네, 1행은 시스템이 무시하므로 자유롭게 한글 라벨을 작성하셔도 됩니다. 데이터는 2행부터 입력하세요.
